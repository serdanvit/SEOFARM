# SEO FARM — простое объяснение «человеческим языком»

## Что это делает

Представьте, что у вас есть 3 «сотрудника-робота»:

1. **Агент 1 (VK-продвижение)**
   - смотрит ваш сайт,
   - придумывает ключевые запросы,
   - создаёт группы VK,
   - публикует контент,
   - постепенно «прогревает» группы.

2. **Агент 2 (лиды)**
   - ищет в VK людей, которые пишут «ищу / посоветуйте / где найти» по вашей теме,
   - отфильтровывает спам,
   - отправляет найденные лиды в Telegram менеджеру.

3. **Агент 3 (статьи)**
   - берёт готовую статью,
   - публикует её на Telegra.ph и в VK-группы,
   - сохраняет историю публикаций.

## Как это работает по шагам

1. Вы указываете сайт и регион.
2. Нажимаете запуск кампании.
3. Система:
   - парсит сайт,
   - определяет нишу,
   - генерирует ключи,
   - ставит задачи в очередь.
4. Планировщик каждые 30 секунд проверяет очередь задач и запускает нужные действия.
5. Результаты видны в таблицах, логах и статистике.

## Как запускать «в одиночку», без программиста

### Минимальный старт

1. Установить зависимости:
   - SQLite режим: `pip install -r requirements.txt`
   - PostgreSQL режим: `pip install -r requirements-postgres.txt`
2. Запустить: `python app.py`
3. Открыть в браузере: `http://localhost:5000`
4. В настройках указать:
   - сайт,
   - регион,
   - бренд.
5. Добавить хотя бы 1 VK-аккаунт через OAuth.
6. Загрузить картинки в `uploads/avatars` и `uploads/covers`.
7. На дашборде запустить кампанию.

### Если что-то «не работает» — куда смотреть

1. **Логи**: раздел Логи в UI или `logs/seofarm.log`.
2. **Планировщик**: `/api/scheduler/status`.
3. **Очередь задач**: `/api/tasks`.
4. **Список API**: `/api/meta/routes`.
5. **Проверка Ollama**: `/api/ai/status`.
6. **Проверка, что фиксы реально применены**: `/api/system/checks`.

### Частые проблемы и быстрые решения

- **Нет лидов**:
  - проверьте активные запросы мониторинга,
  - проверьте токен VK,
  - проверьте Telegram токен/chat_id.

- **Группы не создаются**:
  - нет активных токенов,
  - исчерпан дневной лимит,
  - не заполнены бренд/сайт.

- **Не получается войти через VK (кнопка OAuth)**:
  - в `Настройки` добавьте `public_base_url` (например `http://127.0.0.1:5000` или ваш домен),
  - при необходимости укажите `vk_client_id` (если используете не Kate Mobile),
  - убедитесь, что этот же URL прописан в настройках VK-приложения как допустимый redirect.

- **Wordstat не срабатывает**:
  - нет `yandex_wordstat_token` и `yandex_wordstat_username` в настройках.

- **PostgreSQL не стартует**:
  - установите `pip install -r requirements-postgres.txt`.
